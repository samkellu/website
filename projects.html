DOOM on Arduino

A lightweight version of the original DOOM 3D, written from scratch to run on an ATMega 328p (Arduino). It utilises a basic raycasting shader (I had to scratch that linear algebra itch...) to render the environment onto a SSD1306 OLED screen. This project has evolved over time, from an incredibly basic object renderer, into a first person shooter that is slowly becoming more and more reminiscent of the original.
The game currently supports basic wall textures, movement, depth perception rendering, wall collisions and more, with plans to add enemy NPCs to fight, doors, pickup items etc over the coming year. This project has been my gateway into the wide and complex world of 3D graphics rendering, and is some of the most enjoyable code I've written over the years.

3D Ray-Tracing Shader

Following from my early tests of 3D rendering in Open GL, and later in my arduino rendering projects, I discovered ray tracing. This is a current and ongoing project to develop a photorealistic ray-trace shader in HLSL (Direct X), using the Unity game engine. From the basics of camera projection, ray intersection calculations with basic geometry and tracing to more complex tasks such as mesh rendering, anti aliasing and depth of field, this project has been a blast to work on and has opened my mind to the wonderful world of compute shaders and graphics.

Cards Online

This is a collaborative project between myself and some friends, which aims to create a multiplayer card game using javascript, Two.js and Deno. This is our first step into the world of web full-stack development, and should be a long running project between the four of us. It features a Deno based backend and webserver, and a dynamic and usable Two.js frontend UI. Currently aims to support multiple different "rooms" which can be created and joined by friends via room code, allowing them to play the popular backpacker's card game, "Shithead".

Handwired Keyboard and Firmware

A combined electronics and low-level firware excercise, the creation of this keyboard was an interesting learning experience which left me with a keyboard that I am happy to use on a daily basis. Based on a RP2040 microcontroller, this board uses an array of NK Cream mechanical switches, a custom OLED screen and a modified KBDFans Tiger Lite case to achieve a fun and enjoyable typing experience. Working with the amazing community at QMK was a real eye opener, as they helped me to create and fine tune the firmware for the board, and then pull it to their main repo for release. This whole process helped me to understand the massive amount of attention to detail that is required to build a software product for release into a real and evolving market, as part of a larger community of developers.
